UNIX Filters

UNIX Filters, filter things, as one might expect. S with a filter in the real world you have something that goes through and filter and then comes out as a slightly different, or vastly different, something. Like a water filter for example, you pass some water full of dirt through the filter the dirt gets filtered out and then you have clean water leaving the other end.

That is the basis behind the UNIX filters below. They are little programs that you pass some data into, that data gets changed in some way and then gets output.


So in our case we have:

  • Commands and data (input) which we type into the shell.
  • The fitler of our choosing which has the information to be transformed sent to it via the shell
  • Data (output) which is sent back to the shell by the filter, via a datastream (stdout)

Common and Useful Filters

cat

cat is probably the simpliest filter of all the filter. It outputs its input largely unchanged. If it is given a list of files instead of just one, then it concatenates the files together and then just outputs that.

It's useful command line options include:

  • -n Will add line numbersto the front of every line starting from 1.
  • -s Will reduce consecutive blank lines into a single blank line.
  • -v Will display special control chracters such a new lines and such.

> cat file1          > cat file2
line4                line0
line5                line1
line6                line2
                     line3
                                    
> cat file1 file2    > cat file2 file1
line4                line0
line5                line1
line6                line2
line0                line3
line1                line4
line2                line5
line3                line6

wc

wc is a filter that counts the number of lines, words and letters in its given input. Very useful when used in conjunction with other filters for counting things. The formote of it's output is as seen on the right. The number represent lines, words and characters respectivley and then the file name is printed if the input was a file.

It's useful command line options include:

  • -c Will only display the number of chracters.
  • -w Will only display the number of words.
  • -l Will only display the number of lines.

> wc file1           > wc file2
 3  3 18 file1        4  4 24 file2

> wc file1 file2
 3  3 18 file1
 4  4 24 file2
 7  7 42 total

head/tail

head/tail are filters that outpu the first or last 10 lines of input, respectively. So head will output the first 10, tail will output the last 10. The number of lines can be changed with the below command line option otherwise it defaults to 10. So they will have the same effect on our small example file1 and file2, outputting the whole file, unless the command line option is used.

It's useful command line options include:

  • -x Where x is a number, head/tail will display the first or the last x lines.

> head -1 file1      > head -2 file2
line4                line0
                     line1

> tail -2 file1      > tail -1 file2
line5                line3
line6

sort

sort as it's name suggests takes its input and outputs it in some particular order. By default it will sort lexicographically according the first character on each line. There are also some more advanced features such as being able to sort columnated by any of the columns, the last two command line options enable this feature.

It's useful command line options include:

  • -r Sort in descending order (reverse).
  • -n Sort numerically instead of lexicographically.
  • -d Sort in dictionary order (will ignore non-letters and non-digits).
  • -t'c' Use character c as the column seperator.
  • -k'n' Sort the input using column n.

> cat file1
3
2
5
4
 
> sort file1     sort -r file1
2                   5
3                   4
4                   3
5                   2

uniq

uniq by default removes every copy but one of adjacent identical lines. So that means is you have identical lines in your input but they are distributed uniq would not remove them. A combination of sort and uniq would need to be used.

It's useful command line options include:

  • -c Print out the number of times each line is duplicated.
  • -d Print only lines that were duplicated (will only print one of each dupe).
  • -u Print only lines that were not duplicated (occured uniquely).

> cat file1
cat
dog
mouse
mouse
 
> uniq file1        > uniq -c file1
cat                 1 cat
dog                 1 dog
mouse               2 mouse

> uniq -d file1     > uniq -u file1
mouse               cat
                    dog

cut

cut outputs certain parts of the input lines depending on the command line options specified. it is very useful for manipulating columnated data.

It's useful command line options include:

  • -c(x|x-x|x,x..,x) here x can be any integer and it represents a chracter postion in a line of input data, this will be easier to understand in practice.
  • -f(x|x-x|x,x..,x) here x is the same as above except it represents columns in input data which are spereated by tabs (unless otherwise specified using the next option).
  • -d'c' Sets the character c that seperates the columns in a given input.

> cat file1
hello world
 
> cut -c2 file1     > cut -c1-5 file1
e                   hello

> cut -c1,3-5,8 file1
hlloo

diff

diff compares to input files and then outputs all the lines that are different in each file. It will specifiy if a line appears in file1 but not in file2 and visa versa. This is very usfull for checking the output of a program you might've written against test output data to see if everything is operating as expected.

It's useful command line options include:

  • -i Ignore case.
  • -i Ignore blank lines.

tr

tr or transliterate converts its input character by character into its output according to a given mapping. Use of the command takes the form tr 'sourceChars' 'destinationChars'. The examples will best explain how tr works. Some shorthand can be used such as 'a-z' which corresponds to the entire alphabet.

It's useful command line options include:

  • -c map every chracter not in source chars..
  • -d Delete every character from source chars that occurs.
